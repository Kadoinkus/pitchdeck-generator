# Notso Deck Branding System

## Core Principle
Clean tech + playful character.
Each slide shows only client-facing content.
No internal metadata, prompt text, or debug labels on slides.

## Title Hierarchy
- Use maximum two layers:
  - Kicker: short context line in uppercase (1 line).
  - Sales headline: strong, concise statement.
- Subtitle is optional and only for explanation slides.
- Avoid duplicate title stacks.

## Accent Rule
- Highlight at most one phrase per headline.
- Use accent on high-impact slides:
  - cover
  - opportunity
  - business-impact
  - pricing
  - closing
- Keep all other slide titles neutral for readability.

## Layout Rhythm
- 16:9 canvas only.
- Prefer split composition:
  - text left, visual right.
- Alternate with centered hero where it improves scanning.
- Keep whitespace and panel spacing consistent.

## Image Rules
- Every slide reserves a visual area.
- If no asset exists, show minimal placeholder:
  - icon
  - short helper text
- Character uploads can be mapped to:
  - all mascot slides
  - specific slides (cover, meet-buddy, example-interaction, closing)

## Content Fit Rules
- Cap bullet counts per slide type.
- Keep headline lengths short.
- Use compact card copy to prevent overflow.

## Theme Tokens
Deck visuals are token-driven:
- primaryColor
- accentColor
- secondaryColor
- backgroundColor
- textColor
- headingFont
- bodyFont

No hardcoded client branding should override these tokens.
